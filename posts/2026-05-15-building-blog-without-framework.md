This blog runs on zero frameworks. No React. No Vue. No Svelte. No Next.js. No build step. No bundler. No transpiler. Just three files:

```
index.html
styles.css
script.js
```

That's it. And it works beautifully.

## Why Skip Frameworks?

Modern web development has become a maze of configuration files, dependency trees, and build pipelines. A simple personal blog shouldn't need:

- A `package.json` with 47 dependencies
- A 200-line webpack config
- A CI/CD pipeline to deploy static content
- A build step that takes 30 seconds

## The Pure Files Approach

Every page is hand-crafted HTML. Every style is written in CSS. Every interaction is vanilla JavaScript. The benefits:

1. **Instant understanding** — any developer can read the source without learning a framework
2. **Zero maintenance** — no dependency updates, no security patches for libraries
3. **Instant loading** — no framework to download, no hydration to wait for
4. **Full control** — no abstraction layer between intent and implementation

## What This Proves

You don't need a framework to build something beautiful and functional. You need:

- A clear design direction
- Solid HTML structure
- Intentional CSS
- Purposeful JavaScript

The framework isn't the product. The product is the product.
